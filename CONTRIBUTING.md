## How to Contribute

### Reporting Issues

1. Check existing issues first
2. Use issue templates (bug report, feature request)
3. Include:
   - Hardware/software versions
   - Steps to reproduce
   - Expected vs. actual behavior
   - Relevant logs/screenshots

### Code Contributions

1. **Fork the repository**
2. **Create a feature branch**: `git checkout -b feat/your-feature-name`
3. **Follow coding standards**:
   - C: K&R style, 4-space indent
   - Python: PEP 8, type hints encouraged
   - Verilog: IEEE 1800-2017 SystemVerilog subset
4. **Write tests**: All new code requires tests (pytest for Python, testbenches for Verilog)
5. **Document**: Add docstrings, update README/docs as needed
6. **Commit**: Use conventional commits format:
